# 위플래닛 BARO — 중고 애플 기기 매입가 예측 ML 시뮬레이터

> **"경험과 감"에 의존하던 중고 기기 매입가 산정을, 현장에서 설명 가능한 ML 시스템으로 자동화한 프로젝트입니다.**

4,325건 실거래 데이터를 기반으로 감가상각 → 상태 보정 → 마진 시뮬레이션 3단계 하이브리드 구조를 설계하고, **매입 담당자가 즉시 사용 가능한 모델**을 Streamlit으로 배포했습니다.

## 문제 정의 & 해결 방향

### 문제 정의

위플래닛에서는 **적정 매입가 산정이 경험과 감에 의존**하고 있었습니다. 같은 모델이라도 경과년수·배터리 상태·기스 등급·수리 이력에 따라 가격이 크게 달라지는데, 이를 **체계적으로 반영하는 시스템이 없었습니다**.

### 해결 방향

모델명·연식·사양·상태를 입력하면 **적정 매입가를 자동 산출하는 시스템**을 구축. 위플래닛의 *"바로 적용 가능한 모델"* 요구사항을 반영하여 **현장 설명 가능성과 일관성**을 우선순위에 두었습니다.

## 시스템 파이프라인

```
데이터 수집 → 전처리 → 감가상각 (1단계) → 상태 보정 (2단계) → 마진 산출 (3단계)
  Selenium    정규식    RandomForest       XGBoost+Monotone       전략 3종 시뮬
```

**핵심 설계 원칙**: 단일 모델이 아닌 **2단계 하이브리드 구조**로 설계. *"경과년수 때문에 얼마나 떨어졌고, 상태 때문에 추가로 얼마나 조정됐는지"* 를 분리해서 설명할 수 있어야 매입 담당자가 납득할 수 있기 때문입니다.

## 모델링 과정 (아이패드)

### 1단계: 감가상각 모델

스펙만으로 경과년수별 기본 시세를 산출합니다.

| 항목 | 내용 |
|---|---|
| 타겟 변수 | 감가율 (중고가 / 출시가) |
| 알고리즘 | RandomForest (n_estimators=100, max_depth=10) |
| 입력 피처 | 경과년수, storage_gb, size, 출시가, 세대, 연결방식, 카테고리, 플랫폼 (8개) |
| 전처리 | 카테고리별 IQR 이상치 제거, 잔존가치 0.1~1.0 범위 필터 |
| 성능 | **R² = 0.94**, MAE ≈ 6.2만 원 (5-Fold CV) |
| 예측식 | `예상 중고가 = 출고가 × 감가율` |

**감가율로 타겟을 변환한 이유**: 가격을 직접 예측하면 고가·저가 기기의 스케일 차이 때문에 모델이 편향됩니다. 감가율로 변환하여 **다양한 기종에 범용 적용** 가능하도록 설계했습니다.

### 2단계: 상태 보정 모델

배터리·기스·찍힘 등 제품 상태를 반영하여 최종 시장가를 산출합니다.

| 항목 | 내용 |
|---|---|
| 타겟 변수 | 감가액 (출시가 − 중고가) |
| 알고리즘 | XGBoost (n_estimators=500, max_depth=8) + **Monotone Constraints** |
| 입력 피처 | 경과년수·출시가·카테고리·세대·연결방식·storage·battery·미개봉·기스·풀박스 (10개) |
| 성능 | **R² = 0.75** |
| 검증 | Train/Test 8:2 분할, 5-Fold CV (오차 범위 0.02) |

**Monotone Constraints 적용 이유**: 모델이 배터리 성능 낮은 제품의 가격을 더 높게 예측하거나, 기스가 있는 제품이 깨끗한 제품보다 비싸게 예측되는 **가격 역전 현상**이 발생했습니다. 위플래닛의 *"정교함보다 바로 사용 가능한 모델"* 요구사항에 맞춰, 정확도를 소폭 양보하더라도 **매입 담당자가 신뢰할 수 있는 예측값** 확보를 우선했습니다.

| 피처 | Monotone Constraint | 의미 |
|---|---|---|
| battery | −1 | 배터리 ↑ → 가격 ↑ |
| 미개봉 | −1 | 미개봉 → 가격 ↑ |
| 기스 | +1 | 기스 ↑ → 가격 ↓ |
| 풀박스 | −1 | 풀박스 → 가격 ↑ |

### 3단계: 마진 시뮬레이션

매입 전략별 실질 마진을 자동 산출합니다.

| 전략 | Pro | Air | Mini | Basic | 설명 |
|---|---|---|---|---|---|
| 공격 (재고확보) | 8% | 10% | 13% | 15% | 매입가 높게 → 재고 확보 우선 |
| 기본 (균형) | 10% | 13% | 17% | 20% | 수익과 물량의 균형 |
| 보수 (마진우선) | 12% | 15% | 20% | 25% | 매입가 낮게 → 마진 우선 |

**마진 계산식**:
```
최종매입가  = 판매 예상가 × (1 − 전략별 할인율)
총 비용     = 최종매입가 + 운영비(3만) + 수리비(선택) + 배송비(8천)
순마진      = 판매 예상가 − 총 비용
순마진율    = 순마진 / 판매 예상가 × 100
```

**재고 기간별 가격 하락 시뮬레이션**도 포함 — 위플래닛 실제 데이터 기준 평균 재고 359일, **50.4%가 1년 이상 재고**라는 점을 반영.

### 상세 매입가 측정 모델 성능

| 카테고리 | 모델 | R² | MAE |
|---|---|---|---|
| 맥북 | XGBoost | 0.96 | ±11.4만 원 |
| 아이패드 | XGBoost + Monotone | 0.75 | ±7.9만 원 |

> 맥북은 룰 베이스, 아이패드는 데이터 학습 기반으로 설계하여 R² 차이 발생. 아이패드는 Monotone Constraints 제약으로 정확도를 일부 양보했습니다.

## 방법론 선택의 근거

*"모델이 잘 돌아간다"가 아니라 "왜 이 방법을 선택했는가"를 설명할 수 있어야 합니다.*

| 단계 | 이 프로젝트 선택 | 선택 이유 |
|---|---|---|
| 모델 구조 | 2단계 하이브리드 (감가 → 상태) | 스펙 변화와 상태 변화를 분리해야 매입 담당자가 납득 가능 |
| 감가상각 (1단계) | 타겟을 감가율로 변환 | 감가율로 변환 → 다양한 기종에 범용 적용 가능 |
| 상태 보정 (2단계) | XGBoost + Monotone Constraints | 가격 역전(기스 있는데 더 비쌈) 방지, 현장 신뢰 확보 |
| 상태 정보 추출 | 정규표현식 파싱 → 정형 피처 12개 | 비정형 텍스트는 그대로 쓸 수 없음 → 모델 입력 가능 형태로 재가공 |
| 마진 산출 (3단계) | 전략 3종(공격·기본·보수) + 재고 시뮬레이션 | 고가 모델은 할인율 낮게, 저가 모델은 높게 → 실제 마진 패턴 반영 |

## 문제 해결 경험

### 1. 비정형 텍스트 → 정형 데이터

- **문제**: 매입 접수 시 상태 설명이 일반 텍스트(*"미세기스 있고 배터리 92%"*)로 기록되어 모델 입력으로 사용 불가
- **해결**: 정규표현식 기반 파싱 엔진 구축. 배터리%·기스 등급·찍힘 등급·파손 여부·풀박스·수리 이력·미개봉을 자동 추출. 결측 시 경과년수 기반 배터리 추정 로직 적용
- **결과**: 비정형 텍스트 → **12개 정형 피처**로 변환
- **배운 점**: 텍스트 데이터는 그대로 쓸 수 없다. **재가공을 통해 정형 데이터로 만든 후에 사용**해야 좋은 데이터셋이 된다.

### 2. 모델 예측값의 모순 (가격 역전)

- **문제**: 모델이 배터리 성능 낮은 제품의 가격을 높게 예측하거나, 기스 있는 제품이 깨끗한 제품보다 비싸게 예측되는 등 **비즈니스 상식에 어긋나는 결과** 발생
- **해결**: XGBoost Monotone Constraints를 적용하여 *"배터리 ↑ → 가격 ↑"*, *"기스 ↑ → 가격 ↓"* 등의 방향성을 모델에 강제
- **결과**: 회사에서 요구한 **바로 활용 가능한 모델** 구축, 매입 담당자가 납득 가능한 결과 도출
- **배운 점**: 모델 성능(R²)이 높아도 **현장에서 설명이 안 되면 쓸 수 없다**. 정확도를 소폭 양보하더라도 비즈니스 로직에 맞는 결과를 내는 모델이 실무에서는 더 가치 있다.

## 비즈니스 활용 방안

### 1. 매입·매각 타이밍 판단

- **매각 전략**: 감가 급락 타이밍 이전에 추가 할인으로 빠른 매각 유도 → 현재 악성 재고 50.4% 해소 근거
- **매입 전략**: 제품별 잔존가치 하락 패턴에 따라 수집 타이밍 차등화. iPad Air는 3년 이후 급락 → 2년 이하 매입, MacBook M1은 감가율 완만 → 장기 보유 가능
- **기대효과**: 악성 재고 비율 감소 및 감가 손실 최소화

### 2. 매입 현장 가격의 표준화

- **Before**: 담당자마다 주관적 기준으로 매입가 산정 → 불균형 발생 → 고객 신뢰 하락·이탈
- **After**: 항목별 차감·프리미엄 기준 확립으로 불균형 해소. **신입 담당자도 동일 기준으로 즉시 매입가 산정 가능**
- **기대효과**: 가격 산정 투명성 확보 + 교육 비용 절감 + 매입 전환율 개선

## 실행 방법

### Local 실행

```bash
# 1. 저장소 클론
git clone https://github.com/JungMoonYoung/baro-dashboard2.git
cd baro-dashboard2

# 2. 가상환경 및 의존성
python -m venv venv
# Windows
.\venv\Scripts\activate
# macOS/Linux
source venv/bin/activate
pip install -r requirements.txt

# 3. 대시보드 실행
streamlit run app.py
```
